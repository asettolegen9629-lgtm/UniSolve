# Настройка общей базы данных (бесплатно)

## Вариант 1: Supabase (рекомендуется) - БЕСПЛАТНО

### Шаг 1: Создайте аккаунт на Supabase

1. Перейдите на https://supabase.com
2. Нажмите "Start your project"
3. Войдите через GitHub (бесплатно)

### Шаг 2: Создайте новый проект

1. Нажмите "New Project"
2. Заполните:
   - **Name**: unislove (или любое имя)
   - **Database Password**: придумайте надежный пароль (запомните его!)
   - **Region**: выберите ближайший регион
3. Нажмите "Create new project"
4. Подождите 2-3 минуты пока проект создается

### Шаг 3: Получите строку подключения

1. В проекте Supabase перейдите в **Settings** → **Database**
2. Найдите раздел **Connection string**
3. Выберите **URI** (не Connection pooling)
4. Скопируйте строку подключения (она выглядит так: `postgresql://postgres:[YOUR-PASSWORD]@db.xxxxx.supabase.co:5432/postgres`)

### Шаг 4: Обновите .env файл

Откройте `unislove-backend/.env` и замените `DATABASE_URL`:

```env
DATABASE_URL="postgresql://postgres:ВАШ_ПАРОЛЬ@db.xxxxx.supabase.co:5432/postgres"
PORT=3000
```

**Важно:** Замените `[YOUR-PASSWORD]` на пароль, который вы создали в шаге 2.

### Шаг 5: Примените миграции

```bash
cd unislove-backend
npx prisma db push
node seed.js
```

Готово! Теперь все пользователи будут видеть одни и те же посты!

### Шаг 4: Обновите .env

Замените `DATABASE_URL` в `unislove-backend/.env` на строку из Neon.

## После настройки

После того как вы настроили удаленную БД:

1. **Обновите .env файл** с новой строкой подключения
2. **Примените миграции:**
   ```bash
   cd unislove-backend
   npx prisma db push
   ```
3. **Добавьте тестовые данные (опционально):**
   ```bash
   node seed.js
   ```
4. **Запустите проект:**
   ```bash
   ./start.sh
   ```

Теперь все, кто использует эту же строку подключения, будут видеть одни и те же посты!

---

## Как поделиться с другом

1. Создайте файл `unislove-backend/.env.shared` с вашей строкой подключения:
   ```env
   DATABASE_URL="postgresql://postgres:пароль@db.xxxxx.supabase.co:5432/postgres"
   PORT=3000
   ```

2. Отправьте этот файл другу (через сообщение или безопасно)

3. Друг должен:
   - Скопировать содержимое в свой `unislove-backend/.env`
   - Запустить `./start.sh`

**Или** добавьте `.env.shared` в репозиторий (но НЕ добавляйте `.env` - он в .gitignore)
